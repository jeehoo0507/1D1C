# 1.눈 프레임 
사실 눈의 프레임은 스테레오 뎁스 연구 기준에는 크게 중요할 것 같지는 않다.
하지만 우리가 내는 결론은 로봇틱 분야와 생명공학 분야에 기준을 제시하는 것 임으로 프레임 
또한 하이퍼 파라미터 로 넣어야 한다.

눈의 화질 파트에서 제시할 것처럼 제시한다.

보통 인간의 눈은 연속적인 아날로그 신호로 정보를 받아들이기 때문에 정확히 몇프레임이라고 정확히 말할수 없다.

그렇기에 입력자체는 
1. 60~100사이의 가변적인 주사율로 받지만 이미지 리사이징 하듯이 중간 정보를 떼서 생각하는 경우(이 경우를 나누는 이유는 실시간 스테레오 뎁스의 경우 프레임 즉 과거의 정보도 중요하게 여겨질수 잇기때문이다 . rnn기반이나 일부 트랜스포머 기반)
2. 인간의 뇌는 처리 속도가 있듯이 처리속도에 맞춰 프레임을 일정 수치 이하로 맞춘다

이 두 경우 중 하나를 사용한다


정리
1. 입력 60~100fps(가변적) + 추론 30fps
2. 입력 자체를 모델의 처리 속도에 맞춰서

로봇틱 분야를 생각하면 ==2==번이 좋은 선택이지만 
진화론적 관점이나 생명공학적으로 보았을때에는 ==1==번이 좋은 선택이 된다.


관련 정보 타당성
1. 최소 시간 분해능(한 번에 구분 가능한 가장 짧은 사건)
2. 깜빡임 융합(critical flicker fusion) 기준
3. ‘매끄러운 영상’으로 느끼는 대략 범위
4. 극한 조건에서의 상한선 근사

## 5. 논리적 가변 범위(‘FPS’로 환산한 근사)
1. 사건을 서로 다른 두 시점으로 “구분만 하는” 최소 시간 간격: 약 3–6 ms → 대략 170–330 Hz 수준(운동 자극, 이상 조건)
2. 일반적인 시간 분해능(운동 단서 약함): 약 20–30 ms → 대략 30–50 Hz 수준.[](https://pmc.ncbi.nlm.nih.gov/articles/PMC6802765/)​
3. 깜빡임이 거의 사라져 “연속적인 빛/영상”으로 느끼는 영역: 대략 50–90 Hz 주변, 민감한 경우 100 Hz 이상까지.

- **일상적 영상 체험 기준:** 대략 50–100 Hz 정도에서 연속적·매끄럽다고 느끼는 영역,
    
- **신경·지각 시스템의 잠재적 시간 상한:** 특정 조건에서 수백 Hz까지 감지 가능한 영역,

어차피 로봇틱에서 프레임을 상한없이 쓸수는 없기에 60hz~100hz정도가 적당할듯?

# 2. 눈은 시야각(시차):
사실 트랜스포머 기반 모델 이전의 스테레오 뎁스모델은 오른쪽 왼쪽이미지를 직접 대조하는 식으로 작동햇다 
하지만 트랜스포머의 어텐션 기반 모델인 STTR과 MVS의 경우 별도의 시차각이 특별히 중요하지는 않다 
하지만,카메라가 3,4개일때 카메라의 배치또한  중요해지기 때문에 
시차에 대한 것은 하이퍼파라미터가 아닌 매개변수로 설정해야 할것이다. 




1. **트랜스포머 이전 (CNN 기반 시대)**
"에피폴라 라인(Epipolar Line)이 수평이어야만 한다."

과거의 대표적인 모델들(PSMNet, GC-Net 등)은 **합성곱 신경망(CNN)**을 사용했습니다. 이 방식은 이미지의 특정 좁은 영역(Local Pattern)을 보고 왼쪽/오른쪽 이미지를 매칭합니다.

- **시야각에 대한 제약:**
    
    - CNN 모델은 **Rectification(정류화)** 과정을 거친 이미지를 입력으로 받습니다. 이는 두 카메라의 이미지 높이를 강제로 맞추는 과정입니다.
        
    - **문제점:** 시야각이 넓어질수록(광각, 어안 렌즈) 이미지 가장자리의 **방사형 왜곡(Radial Distortion)**이 심해집니다. 이를 억지로 평면으로 펴서 정류화하면, 가장자리 정보가 날아가거나 해상도가 급격히 떨어집니다.
        
    - **결과:** 그래서 보통 좁은 시야각(일반적인 웹캠이나 자율주행 전방 카메라 수준)에 최적화되어 있었고, 넓은 시야각을 처리하려면 이미지를 잘라내서 써야 했습니다.

2. **현재 (트랜스포머 및 최신 기술 도입 후)**
"전역적 문맥(Global Context) 이해와 구면 기하학의 도입"

---

#### A. 트랜스포머의 어텐션(Attention) 메커니즘

트랜스포머는 이미지의 국소적인 부분만 보는 게 아니라, **이미지 전체의 연관성(Global Context)을 봅니다.

- **시야각과의 관계:** 광각 렌즈의 가장자리는 왜곡이 심하거나, 텍스처가 늘어져서 매칭이 어렵습니다. 트랜스포머는 이미지의 중앙부(정보가 확실한 곳)의 정보를 참고하여 가장자리의 깊이 정보도 더 정확하게 추론할 수 있습니다.
    

#### B. 구면 스테레오(Spherical Stereo) 및 360도 처리

단순히 모델 구조만 바뀐 것이 아니라, **기하학적 접근 방식**이 바뀌었습니다.

- **평면(Planar)에서 구면(Spherical)으로:** 이제는 180도 이상의 어안 렌즈나 360도 카메라의 이미지를 억지로 평면으로 펴지 않고, **구면 좌표계(Spherical Coordinate)** 상에서 직접 깊이를 추정하는 연구가 활발합니다.
    
- **적응형 방식:** 최신 모델들은 렌즈의 왜곡 파라미터 자체를 학습하거나, 왜곡된 상태에서도 매칭이 가능한 유연한 네트워크 구조를 가집니다.

# 연구 결과 예측 및 구성

**Introduction:**

- 왜 인간은 눈이 2개인가? (진화론적 관점)
    
- 로봇은 2개여야 하는가? 3, 4개면 더 좋은가?
    
- 딥러닝(Transformer)을 이용해 이를 시뮬레이션하고 최적해를 찾겠다.

**Methodology:**

- **Architecture:** N-View Transformer Model 설명. (입력 개수 가변적)
    
- **Bio-mimicry:** 인간 시각 파라미터(Baseline, Foveation) 적용 방법.
    
- **Efficiency Metric:** 위에서 제안한 수식($L_{total}$) 정의.

**Experiments (핵심):**

- **Case 1 (N=2):** 기준점. Occlusion 영역에서의 에러율 측정.
    
- **Case 2 (N=3):** "Tri-focal Tensor" 효과. Occlusion이 얼마나 획기적으로 줄어드는가? 연산량은 얼마나 느는가?
    
- **Case 3 (N=4+):** 정확도 상승폭이 둔화되는 구간 확인 (Diminishing Returns).

**Discussion:**

- 결과 분석: "인간이 눈이 2개인 이유는 3개째부터 얻는 정보 이득보다 뇌가 쓰는 에너지가 더 크기 때문으로 추정된다."
    
- 로봇 적용: "하지만 배터리가 넉넉하고 정밀 조작이 필요한 로봇 팔(Arm)의 경우 N=3이 최적일 수 있다."

**Conclusion:**

- 수식($L_{total}$)을 통한 센서 구성의 가이드라인 제시
