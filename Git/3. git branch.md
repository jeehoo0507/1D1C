- 브랜치 목록 확인
```bash
git branch 
```

- 브랜치 생성
```bash
git branch 브랜치이름
```

- 브랜치 변경
```bash
git switch 브랜치이름
```

- 브랜치 이름 변경
```bash
git branch -m 새이름           # 지금 서 있는 브랜치 이름 변경
git branch -m 옛이름 새이름     # 특정 브랜치 이름 변경 (지금 안 서 있어도 됨)
```

- 특정 커밋을 가리키는 브랜치 만들기
```bash
git branch 브랜치이름 <커밋해시>
```




- 브랜치를 main과 합치기
```bash
git switch 브랜치이름
```

```bash
git merge 브랜치이름
``` 

만약 브랜치에서도 a파일을 수정하고 main에서도 a파일을 수정했다면 오류가 남 
해결하고 나서 add+commit하면 됨

만약 수정 안하고 add+commit 하면 오류난 걸 알려주는 파일 그대로 올라감


- 브랜치 삭제
1. merge를 이미 한 경우
```bash
git branch -d 브랜치이름
```
안전 삭제

2. merge를 안 한 경우
```bash
git branch -D 브랜치 이름
```
강제 삭제
