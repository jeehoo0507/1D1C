### Git 설치
```bash
sudo apt install git -y
```

 
#### 설정
```bash
git config --global user.name "이름"
git config --global user.email "이메일"
```
global을 통해 로컬 전체에 대한 세팅을 하는 것
무조건 github에서 쓰는 아이디와 이메일을 사용하지 않아도 됨


- 컴퓨터 하나로 작업할 때
```bash
cd 폴더 이름 # 폴더 터미널로 열면 굳이 안 해도 됨
git config user.name "이름"
git config user.email "이메일"
```


서버컴처럼 하나의 컴퓨터에 여러명이 작업하는 상황이라면 글로벌 설정 이외에 자신이 작업하는 폴더 내에서 따로 아이디와 이메일이 설정 가능함
글로벌 설정 위에 덮어쓰는 방식


- 확인 방법
```bash
# 글로벌 설정 확인
git config --global user.name
git config --global user.email

# 폴더 내 설정 확인
git config user.name
git config user.email

# 로컬로 등록되어있는지 확인(없으면 빈칸 출력)
cd 폴더 이름
git config --local user.name
git config --local user.email
```

- 초기화

폴더 내부 초기화
```bash
git config --unset user.name
git config --unset user.email
```
 이 경우 글로벌 설정이 있다면 글로벌 설정으로 변경됨(덮어씌운 로컬 설정 초기화했으니까)

글로벌 설정 초기화
```bash
git config --global --unset user.name
git config --global --unset user.email
```

- 확인 방법
```bash
git config --list # 현재 적용 중인 모든 설정 (로컬+글로벌) 
git config --list --local # 이 폴더 로컬 설정만 
git config --list --global # 글로벌 설정만
```

 `git init`을 통해 git이 적용 된 폴더에서 `git add`를 통해 대기열에 올리고, `git commit`을 통해 레포에 올리는 것이 가능함

#### git add
```bash
git add . # 현재 폴더 + 하위 폴더 전체
git add 파일명.확장자 # 특정 파일 하나만
git add 폴더명/ # 특정 폴더 전체
git add *.확장자 # 특정 확장자를 가진 파일만
```


### Unstage

git add를 통해 대기열에 올린 것들을 내리는 것도 가능함(add 와 동작 자체는 비슷함)
```bash
git restore --staged 파일명.확장자
git restore --staged 폴더명/
git restore --staged .
```

- 첫 커밋 전
커밋을 한 번도 안 한 상태에서는 아래 명령어를 사용해야함(위에 명령어 자체가 커밋 전으로 되돌리기라는 뜻이기 때문)
```bash
# 단일 파일
git rm --cached 파일명

# 단일 파일 아님
git rm --cached -r 폴더명/ 
git rm --cached -r .

```
 첫 커밋 전 대기열에 올라온 내용을 삭제하는 과정은 파일 자체를 지우기 때문에 단일 파일 삭제가 아닌 경우
 `-r` 재귀 옵션을 사용해야함

### git status
```bash
git status
```

`git status` 는 지금 각 파일이 어느 단계에 있는지 보여주는 명령어

- **Untracked** — git이 아예 모르는 새 파일
- **Not staged** — 수정은 했지만 add 안 함
- **Staged (to be committed)** — add 해서 커밋 대기 중

### git log
- 기본
```bash
git log
```
전체 커밋 히스토리를 자세히 보여줌 (커밋 해시, 작성자, 날짜, 메시지)

- 한 줄 씩 간단하게
```bash
git log --oneline
```

```bash
# 출력 예시
a1b2c3d 기능 추가
e4f5g6h 버그 수정
h7i8j9k first commit
```


- 특정 파일
```bash
# 한 파일
git log 파일명.확장자
git log --oneline 파일명.확장자

# 폴더
git log 폴더명/                    
git log --oneline 폴더명/
git diff           # 스테이징된 "hello" ↔ 지금 "hello world"  (add 이후의 추가 변경분)
git diff --staged  # 마지막 커밋 "hi" ↔ 스테이징된 "hello"     (커밋 이후 add된 변경분)
# 현재 폴더 전체
git log .
git log --oneline .  
```


- `--all` vs  `.`
```bash
git log . # 현재 위치 폴더의 커밋만 필터링(즉, 현재 스위치 된 브랜치)
git log --all # 현재 위치와 상관없이 레포 안 모든 브랜치의 커밋 
```

branch에 대해선 다음에 알아보자

### git commit
- 기본
```bash
git commit -m "커밋 메시지"
```
스테이징(add된) 상태를 하나의 기록으로 확정`-m`은 메시지를 바로 붙이는 옵션임

- add+commit 한번에
```bash
git commit -am "메시지"
```

추적 중인 것만 됨(Untracked 파일이면 안됨)


- 커밋 취소(reset)
```bash
git reset --soft HEAD~1     # 커밋만 취소, 변경 내용은 스테이징 상태로 되돌림
git reset --mixed HEAD~1    # 커밋+스테이징 취소, 파일 수정 내용은 남음 (기본값)
git reset --hard HEAD~1     # 커밋+수정내용까지 완전 삭제
```
`HEAD~1` = 1개 전 커밋 기준
1,2,3.... 다 됨


- 이미 push 한 커밋 되돌리기(revert)
```bash
git revert <커밋해시>
```
기록은 안 지우고 "되돌리는 새 커밋"을 추가함



- 레포에서 파일 지우기

- 로컬 파일도 삭제
```bash
git rm 파일명
git commit -m "파일 삭제"
```
rm이 로컬에서 지움 + add해서 스테이징(즉, add 안해도 됨)


- 로컬 파일은 보존
```bash
git rm --cached 파일명
git commit -m "레포에서 제외"
git push
```

폴더나 여러 파일 한꺼번에 지울 땐 `-r` 필요함
```bash
git rm --cached -r 폴더명/
```
