```python
from pathlib import Path

path = Path("폴더경로")   

for p in sorted(path.iterdir()):
    kind = "폴더" if p.is_dir() else "파일"
    print(kind, "|", p.name)
```


1. Path 라이브러리 가져오기
```python
from pathlib import Path
```

2. path 저장하기
```python
path = Path("폴더경로")  
```


 
#### Path를 써야하는 이유

기존 문자열 방식
```python
# 문자열 방식
path = "/root/.cache/kagglehub/datasets/.../1"
cat_folder = path + "/Cat"          # 슬래시 직접 챙겨야 함
img = path + "/Cat" + "/0.jpg"      # 점점 지저분
```

Path 방식
```python
# Path 방식
path = Path("/root/.cache/kagglehub/datasets/.../1")
cat_folder = path / "Cat"           # 슬래시 알아서 처리
img = path / "Cat" / "0.jpg"        # 읽기도 편함
```

`/` 연산자가 슬래시를 알아서 챙겨줌



3. 폴더 안 항목 훑기
```python
for p in sorted(path.iterdir()):
```

`path.iterdir()`
`path` 폴더 바로 밑에 있는 항목들을 하나씩 꺼내줌
하위 폴더 속까지 파고들지 않고 딱 한 겹만. 여기서 나오는 `p` 하나하나도 전부 Path 객체

```
path/                    ← 여기가 기준
├── Cat/                 ← iterdir()가 꺼내는 건 여기까지 (한 겹)
│   ├── 0.jpg            ← 이 안쪽은 안 봄
│   └── 1.jpg
└── Dog/                 ← 여기까지
    ├── 0.jpg            ← 이 안쪽도 안 봄
    └── 1.jpg
```


`sorted(...)`
그 항목들을 이름 순으로 정렬해. 안 쓰면 순서가 뒤죽박죽으로 나올 수 있어서 편리함


4. 폴더? 파일?
```python
if p.is_dir():
    kind = "폴더"
else:
    kind = "파일"
```

`p.is_dir()`은 "이 항목이 폴더냐?"를 묻고 참/거짓을 돌려줌

```python
print(kind, "|", p.name)
```

`p.name`은 전체 경로가 아니라 **맨 끝 이름만** 뽑아줘 (예: `/root/.../Cat` → `Cat`)
