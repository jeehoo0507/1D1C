#### 라이브러리
```python
import torch
from torch.utils.data import random_split, DataLoader
from torchvision import datasets, transforms
```

`torch` -> pytorch
`random_split` -> 데이터 쪼개기
`DataLoader` -> 데이터를 배치로 묶기

`datasets`/`transforms` -> 이미지 불러옴 & 변환


#### 이미지 변환 정의
```python
transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
])
```

이미지를 모델에 넣기 전에 어떻게 손볼지 규칙을 정하는부분
`Compose`는 "이 작업들을 순서대로 해라"라는 묶음이다.

위 코는 아래 두 가지 일을 한다.

##### 1.
`Resize((224, 224))` - 모든 이미지를 가로세로 224픽셀로 크기 통일. 원본 이미지는 크기가 제각각인데, 모델은 같은 크기만 받으니까 맞춰주는 거임

##### 2.
`ToTensor()` - 이미지를 텐서(숫자 덩어리)로 변환 컴퓨터는 그림을 못 다루고 숫자만 다루니까, 픽셀을 숫자 배열로 바꾸는거임 이때 픽셀값도 0~255에서 0~1 범위로 자동 조정


#### 데이터 불러오기 + 라벨링
```python
full = datasets.ImageFolder(root="폴더 경로", transform=transform)
```

지정한 폴더 바로 하위 폴더들을 클래스(라벨)로 두고 transform 까지 적용


#### 90:10 분할
```python
n_train = int(len(full) * 0.9)
train_ds, test_ds = random_split(full, [n_train, len(full) - n_train])
```

`len(full)`은 전체 이미지 개수. 거기에 0.9를 곱하고 `int`로 소수점 버려서 train에 쓸 개수를 구함
예를 들어 전체가 1000장이면 `n_train = 900`

`random_split`은 `full`을 `[900, 100]`으로 랜덤하게 쪼개. `len(full) - n_train`이 나머지(100)라서 항상 딱 떨어짐
결과로 `train_ds`(900장), `test_ds`(100장) 두 덩어리가 나와. 파일 복사 없이 인덱스만 나눔

#### 배치로 묶기
```python
train_loader = DataLoader(train_ds, batch_size=32, shuffle=True)
test_loader  = DataLoader(test_ds, batch_size=32)
```

`DataLoader`는 데이터를 32개씩 묶음(batch)으로 꺼내주는 도구
모델은 이미지를 한 장씩이 아니라 여러 장 묶어서 학습하는 게 효율적임

`shuffle=True`(train만) - 매번 순서를 섞임.
안 그러면 모델이 "Cat 다음엔 항상 Cat"처럼 순서를 외워버려서 학습이 왜곡돼. test는 순서 상관없어서 셔플 안함
